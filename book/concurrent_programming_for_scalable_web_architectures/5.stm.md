STM 和多线程一样使用 shared state 和 lock。
但是 STM 不需要手动控制锁，提供了高一级的抽象。

---


---

## application logic

+ 和线程模型一样，每个请求对应一个线程。
+ 之前提过线程模型不适合 IO 密集型的请求，STM 干脆禁止在 transaction 中进行 IO 操作
+ STM 不需要手动控制锁，使得共享数据变得更简单可控，但是没能解决并行 IO 的问题
